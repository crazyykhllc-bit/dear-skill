# 对话纠正处理

## 任务
当用户在对话中说"ta不会这样说"/"不对"/"ta应该是"时，识别纠正内容，更新对应文件。

---

## 识别触发词
- "不对" / "错了" / "不像"
- "ta不会这样说" / "ta不会这么做"
- "ta应该是" / "ta其实是"
- "这不是ta的风格"
- "实际上ta会……"

## 分类规则

| 纠正内容 | 更新文件 | 更新位置 |
|---------|---------|---------|
| "ta不会这样说话" / 说话方式相关 | persona.md | Layer 2 或 Correction 记录 |
| "ta不会这样关心人" / 关心方式相关 | care.md | 对应章节或 Correction 记录 |
| "这件事不是这样的" / 事实性修正 | memory.md | 对应章节 |
| "ta的价值观不是这样" / 观点态度 | persona.md | Layer 3 |
| "ta生气的时候不会这样" / 情绪反应 | persona.md | Layer 3 |

## 生成格式

在对应文件的 `## Correction 记录` 节追加：

```markdown
### Correction #{n} — {日期}
- 用户说："{用户原话}"
- 原始内容：{被纠正的内容}
- 修正为：{正确表述}
- 影响范围：{简述对哪些场景会改变输出}
```

## 执行流程

1. 识别用户的纠正意图
2. 定位被纠正的内容在哪个文件的哪个位置
3. 生成 Correction 记录
4. 追加到对应文件
5. 重新生成 SKILL.md
6. 回应用户："好的，已经更新了。{简述改了什么}"

## 注意事项

- Correction 的优先级高于原始内容
- 如果多次 Correction 互相矛盾，以最新一次为准
- 纠正时语气要温和："你说得对，我调整一下"，不要质疑用户的记忆
